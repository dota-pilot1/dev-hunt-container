React + STOMP

connect 함수 집중 설명

이 문서는 ChatStudyLevel1Page.tsx의 connect() 함수만 집중해서 설명한다. 이 함수의 목적은 STOMP client를 만들고, WebSocket에 연결하고, 연결 성공 후 topic을 구독하는 것이다.

전체 코드

const connect = () => {
  if (clientRef.current?.active) return;

  setConnectionState("connecting");
  appendLog("system", `STOMP 연결 시도: ${wsUrl}`);

  const client = new Client({
    brokerURL: wsUrl,
    reconnectDelay: 0,
    debug: (line) => {
      if (process.env.NODE_ENV === "development") {
        console.debug("[stomp]", line);
      }
    },
    onConnect: () => {
      setConnectionState("connected");
      appendLog("system", "STOMP 연결 완료");
      client.subscribe("/topic/chat.level1.pong", (frame: IMessage) => {
        const body = JSON.parse(frame.body) as PongMessage;
        appendLog(
          "received",
          `${body.sender}: ${body.message} (${new Date(body.serverTime).toLocaleTimeString("ko-KR")})`
        );
      });
    },
    onDisconnect: () => {
      setConnectionState("disconnected");
      appendLog("system", "STOMP 연결 해제");
    },
    onStompError: (frame) => {
      setConnectionState("error");
      appendLog("error", frame.headers.message ?? "STOMP 프로토콜 에러");
    },
    onWebSocketError: () => {
      setConnectionState("error");
      appendLog("error", "WebSocket 연결 에러");
    },
    onWebSocketClose: () => {
      if (connectionState !== "idle") {
        setConnectionState("disconnected");
      }
    },
  });

  clientRef.current = client;
  client.activate();
};

한 줄 요약

connect()는 이미 연결 중이면 중복 실행을 막고, 새 STOMP Client를 만든 다음, client.activate()로 실제 WebSocket 연결을 시작한다. 연결이 성공하면 /topic/chat.level1.pong을 구독한다.

실행 순서

1. 중복 연결 방지 clientRef.current?.active가 true면 바로 종료한다.
2. 상태 변경 화면 상태를 connecting으로 바꾸고 로그를 남긴다.
3. Client 생성 new Client({...})로 STOMP client 설정을 만든다.
4. ref에 보관 clientRef.current = client로 컴포넌트 생명주기 동안 보관한다.
5. 연결 시작 client.activate()가 실제 연결을 시작한다.
6. 구독 등록 연결 성공 후 onConnect에서 topic을 구독한다.

부분별 설명

1. 중복 연결 방지

if (clientRef.current?.active) return;

이미 STOMP client가 활성 상태라면 다시 연결하지 않는다. 사용자가 연결 버튼을 여러 번 눌러도 WebSocket 연결이 중복으로 열리지 않게 막는 코드다.

값 의미
clientRef.current 현재 컴포넌트가 보관 중인 STOMP client 인스턴스
active client가 활성화됐거나 연결 시도 중인지 나타내는 값
return 이미 활성 상태면 함수 실행 중단

2. 연결 중 상태 표시

setConnectionState("connecting");
appendLog("system", `STOMP 연결 시도: ${wsUrl}`);

사용자가 연결 버튼을 눌렀다는 것을 화면에 바로 반영한다. 실제 연결 성공 여부와 별개로, 현재 상태가 연결 시도 중임을 보여준다.

  • setConnectionState("connecting"): 상단 상태 배지를 연결 중으로 변경
  • appendLog(...): 이벤트 로그에 연결 시도 기록 추가
  • wsUrl: 실제 접속할 WebSocket 주소. 기본값은 ws://localhost:4101/ws

3. STOMP Client 생성

const client = new Client({
  brokerURL: wsUrl,
  reconnectDelay: 0,
  debug: ...
  onConnect: ...
  onDisconnect: ...
  onStompError: ...
  onWebSocketError: ...
  onWebSocketClose: ...
});

이 시점에는 아직 연결이 열린 것이 아니다. new Client는 연결에 필요한 설정과 이벤트 핸들러를 가진 객체를 만드는 단계다.

실제 연결은 맨 아래의 client.activate()가 실행될 때 시작된다.

4. brokerURL

brokerURL: wsUrl,

STOMP client가 접속할 WebSocket endpoint다. 백엔드의 WebSocketConfig에서 등록한 endpoint와 맞아야 한다.

프론트 백엔드
ws://localhost:4101/ws registry.addEndpoint("/ws")

5. reconnectDelay

reconnectDelay: 0,

자동 재연결을 끈 설정이다. Level 1에서는 연결 성공/실패를 직접 눈으로 확인하는 것이 목적이라 자동 재연결을 꺼두는 편이 이해하기 쉽다.

나중에 실전 채팅에서는 reconnectDelay: 3000처럼 설정해서 연결이 끊겼을 때 자동으로 다시 붙게 만들 수 있다.

6. debug

debug: (line) => {
  if (process.env.NODE_ENV === "development") {
    console.debug("[stomp]", line);
  }
},

STOMP 내부 로그를 브라우저 콘솔에 출력한다. 개발 환경에서만 출력하게 막아두었기 때문에 운영 빌드에서는 불필요한 로그가 나오지 않는다.

  • CONNECT 프레임
  • CONNECTED 프레임
  • SUBSCRIBE 프레임
  • SEND 프레임
  • DISCONNECT 프레임

7. onConnect

onConnect: () => {
  setConnectionState("connected");
  appendLog("system", "STOMP 연결 완료");
  client.subscribe("/topic/chat.level1.pong", (frame: IMessage) => {
    const body = JSON.parse(frame.body) as PongMessage;
    appendLog(
      "received",
      `${body.sender}: ${body.message} (${new Date(body.serverTime).toLocaleTimeString("ko-KR")})`
    );
  });
},

WebSocket 연결과 STOMP CONNECT 과정이 모두 성공하면 실행된다. 이 안에서 상태를 connected로 바꾸고, 서버 응답 topic을 구독한다.

왜 구독을 onConnect 안에서 하나?

STOMP 구독은 연결이 성공한 뒤에 해야 안정적이다. 연결 전에는 서버와 세션이 없기 때문에 구독 프레임을 보낼 수 없다.

구독 주소

client.subscribe("/topic/chat.level1.pong", callback)

백엔드의 @SendTo("/topic/chat.level1.pong")과 같은 주소다. 서버가 이 topic으로 메시지를 보내면 이 callback이 실행된다.

8. frame.body 파싱

const body = JSON.parse(frame.body) as PongMessage;

STOMP로 받은 메시지는 IMessage 객체로 들어오고, 실제 서버 응답 JSON 문자열은 frame.body에 들어 있다.

필드 의미
frame STOMP 메시지 전체 객체
frame.body 서버가 보낸 JSON 문자열
PongMessage { sender, message, serverTime } 타입

9. onDisconnect

onDisconnect: () => {
  setConnectionState("disconnected");
  appendLog("system", "STOMP 연결 해제");
},

STOMP 연결이 정상적으로 해제되면 실행된다. 예를 들어 사용자가 연결 해제 버튼을 눌러 client.deactivate()를 호출하면 이 흐름을 탄다.

10. onStompError

onStompError: (frame) => {
  setConnectionState("error");
  appendLog("error", frame.headers.message ?? "STOMP 프로토콜 에러");
},

WebSocket 연결 자체는 열렸지만 STOMP 프로토콜 레벨에서 에러가 발생하면 실행된다. 예를 들어 broker 처리 중 문제가 있거나 서버가 ERROR 프레임을 보내는 경우다.

11. onWebSocketError

onWebSocketError: () => {
  setConnectionState("error");
  appendLog("error", "WebSocket 연결 에러");
},

WebSocket 연결 자체에서 에러가 났을 때 실행된다. 서버가 꺼져 있거나, endpoint가 틀렸거나, CORS/handshake 문제가 있을 때 확인해야 하는 지점이다.

12. onWebSocketClose

onWebSocketClose: () => {
  if (connectionState !== "idle") {
    setConnectionState("disconnected");
  }
},

WebSocket이 닫히면 실행된다. 정상 해제든 서버 종료든 소켓이 닫히면 화면 상태를 disconnected로 맞춘다.

이 코드는 현재 렌더 시점의 connectionState를 클로저로 잡는다. 실전에서는 최신 상태 참조가 필요하면 useRef로 상태를 같이 관리하거나, close 이벤트에서는 단순히 disconnected로 바꾸는 방식이 더 명확할 수 있다.

13. clientRef.current = client

clientRef.current = client;

생성한 STOMP client를 React ref에 저장한다. ref에 저장하는 이유는 렌더링 사이에서도 같은 client 인스턴스를 유지해야 하기 때문이다.

  • 연결 해제 버튼에서 clientRef.current.deactivate() 호출 가능
  • Ping 전송 버튼에서 clientRef.current.publish() 호출 가능
  • 컴포넌트 unmount 시 연결 정리 가능

14. client.activate()

client.activate();

실제 연결을 시작하는 코드다. 이 줄이 실행되면 브라우저가 brokerURL로 WebSocket 연결을 열고, 그 위에서 STOMP CONNECT 프레임을 보낸다.

연결이 성공하면 onConnect가 실행되고, 실패하면 onWebSocketError 또는 onStompError가 실행될 수 있다.

연결 성공 시 실제 흐름

  1. 사용자가 연결 버튼 클릭
  2. connect() 실행
  3. new Client(...)로 STOMP client 생성
  4. client.activate() 실행
  5. 브라우저가 ws://localhost:4101/ws로 WebSocket 연결
  6. STOMP CONNECT 프레임 전송
  7. 서버가 CONNECTED 프레임 응답
  8. onConnect 실행
  9. /topic/chat.level1.pong 구독 등록
  10. 화면 상태가 연결됨으로 변경

백엔드 코드와 매칭

프론트 코드 백엔드 코드 의미
brokerURL: wsUrl registry.addEndpoint("/ws") WebSocket 접속 주소
client.subscribe("/topic/chat.level1.pong", ...) @SendTo("/topic/chat.level1.pong") 서버 응답을 받을 구독 주소
client.publish({ destination: "/app/chat.level1.ping" }) @MessageMapping("/chat.level1.ping") 클라이언트가 서버 메서드로 메시지를 보내는 주소

이 함수에서 중요한 포인트

  • new Client는 client 객체 생성이고, 실제 연결은 activate()다.
  • 구독은 onConnect 안에서 해야 한다.
  • clientRef는 STOMP client를 렌더링과 무관하게 유지하기 위한 저장소다.
  • frame.body는 JSON 문자열이므로 JSON.parse가 필요하다.
  • WebSocket 에러와 STOMP 에러는 서로 다른 층의 에러다.
  • Level 1에서는 자동 재연결을 꺼서 연결 상태 변화를 직접 확인한다.
