ws 컨트롤러 구조 분석

이 문서는 Level 1 WebSocket/STOMP 컨트롤러인 ChatStudyMessageController의 구조를 분석한다. 핵심은 @MessageMapping으로 메시지를 받고, @SendTo로 구독자에게 응답하는 흐름이다.

대상 파일

dev-hunt-server/src/main/java/com/cj/devhunt/chat/presentation/ChatStudyMessageController.java

전체 코드

package com.cj.devhunt.chat.presentation;

import com.cj.devhunt.chat.presentation.dto.ChatLevel1PingRequest;
import com.cj.devhunt.chat.presentation.dto.ChatLevel1PongResponse;
import org.springframework.messaging.handler.annotation.MessageMapping;
import org.springframework.messaging.handler.annotation.SendTo;
import org.springframework.stereotype.Controller;

import java.time.Instant;

@Controller
public class ChatStudyMessageController {

    @MessageMapping("/chat.level1.ping")
    @SendTo("/topic/chat.level1.pong")
    public ChatLevel1PongResponse ping(ChatLevel1PingRequest request) {
        String sender = request.sender() == null || request.sender().isBlank()
                ? "anonymous"
                : request.sender().trim();
        String message = request.message() == null || request.message().isBlank()
                ? "ping"
                : request.message().trim();

        return new ChatLevel1PongResponse(sender, "pong: " + message, Instant.now());
    }
}

이 클래스가 Spring이 관리하는 컨트롤러 Bean임을 나타낸다. REST API 컨트롤러에서는 보통 @RestController를 쓰지만, STOMP 메시지 컨트롤러에서는 @Controller를 사용한다.

@RestController는 HTTP 응답 body를 직접 반환하는 REST용 성격이 강하다. 여기서는 HTTP 응답이 아니라 STOMP broker로 메시지를 보내는 구조라 @Controller가 자연스럽다.

@MessageMapping

@MessageMapping("/chat.level1.ping")

프론트가 보낸 STOMP 메시지를 이 메서드로 라우팅하는 주소다.

프론트는 실제로 아래 주소로 메시지를 보낸다. 

client.publish({
  destination: "/app/chat.level1.ping",
  body: JSON.stringify(payload),
});

그런데 백엔드에서는 @MessageMapping("/chat.level1.ping")이다. 이유는 /app prefix가 설정에서 서버 애플리케이션 prefix로 등록되어 있기 때문이다. 

registry.setApplicationDestinationPrefixes("/app");
프론트 발행 주소 Spring 처리 컨트롤러 매핑
/app/chat.level1.ping /app prefix 제거 @MessageMapping("/chat.level1.ping")

@SendTo

@SendTo("/topic/chat.level1.pong")

이 메서드의 반환값을 어디로 보낼지 정하는 어노테이션이다. 여기서는 반환값을 /topic/chat.level1.pong destination으로 보낸다.

프론트는 이 topic을 미리 구독하고 있다. 

client.subscribe("/topic/chat.level1.pong", callback)

따라서 ping() 메서드가 값을 반환하면, Spring simple broker가 /topic/chat.level1.pong 구독자들에게 메시지를 전달한다.

@SendTo는 요청한 사람에게만 보내는 것이 아니다. 해당 topic을 구독 중인 모든 클라이언트에게 전송된다.

이 메서드는 프론트가 보낸 JSON 메시지를 ChatLevel1PingRequest로 받아 처리한다.

부분 의미
ChatLevel1PingRequest request 프론트가 보낸 JSON body를 Java record로 변환한 값
String sender 보낸 사람 이름. 비어 있으면 anonymous 사용
String message 보낸 메시지. 비어 있으면 ping 사용
return new ChatLevel1PongResponse(...) 프론트 구독자에게 보낼 응답 객체 생성

요청 DTO

프론트가 보내는 JSON:

{
  "sender": "tester",
  "message": "hello websocket stomp"
}

백엔드 요청 DTO:

public record ChatLevel1PingRequest(
        String sender,
        String message
) {
}

Spring 메시지 컨버터가 JSON body를 이 record로 변환해서 ping(ChatLevel1PingRequest request)에 넣어준다.

응답 DTO

백엔드 응답 DTO:

public record ChatLevel1PongResponse(
        String sender,
        String message,
        Instant serverTime
) {
}

프론트에서 받는 JSON 형태:

{
  "sender": "tester",
  "message": "pong: hello websocket stomp",
  "serverTime": "2026-06-27T..."
}

이 응답은 HTTP response로 돌아가는 것이 아니라, @SendTo("/topic/chat.level1.pong")에 의해 topic 메시지로 전송된다.

REST 컨트롤러와 비교

구분 REST STOMP WebSocket
클래스 @RestController @Controller
요청 매핑 @PostMapping("/api/...") @MessageMapping("/...")
클라이언트 요청 axios.post(...) client.publish(...)
응답 HTTP response body @SendTo topic 메시지
클라이언트 수신 await 결과 subscribe callback

핵심 매칭

역할 프론트 백엔드
요청 보내기 publish("/app/chat.level1.ping") @MessageMapping("/chat.level1.ping")
응답 받기 subscribe("/topic/chat.level1.pong") @SendTo("/topic/chat.level1.pong")

정리

  • @MessageMapping은 프론트가 publish한 메시지를 받는 주소다.
  • @SendTo는 메서드 반환값을 보낼 topic 주소다.
  • 프론트는 /app/...으로 보내고, 백엔드는 /app을 뺀 주소로 받는다.
  • 프론트는 /topic/...을 구독하고, 백엔드는 같은 topic으로 응답을 보낸다.
  • @SendTo 응답은 해당 topic 구독자 전체에게 전달된다.
