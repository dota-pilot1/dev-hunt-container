Level 1

WebSocket STOMP 연결 로직 정리

목표는 브라우저와 Spring Boot 서버가 STOMP over WebSocket으로 연결되고, 프론트에서 ping을 보내면 서버가 pong을 topic으로 브로드캐스트하는 최소 흐름을 검증하는 것이다.

전체 흐름

  1. 프론트가 ws://localhost:4101/ws로 STOMP 연결을 연다.
  2. 연결 성공 후 프론트가 /topic/chat.level1.pong을 구독한다.
  3. 사용자가 Ping 버튼을 누르면 프론트가 /app/chat.level1.ping으로 메시지를 발행한다.
  4. 백엔드의 @MessageMapping("/chat.level1.ping") 메서드가 메시지를 받는다.
  5. 백엔드는 응답 DTO를 만들어 /topic/chat.level1.pong으로 전송한다.
  6. 프론트 구독 콜백이 pong 메시지를 받아 이벤트 로그에 출력한다.

프로세스별 관련 파일

2. 보안 허용 dev-hunt-server/src/main/java/com/cj/devhunt/config/SecurityConfig.java /ws, /ws/** 요청을 permitAll 처리. WebSocket handshake가 Spring Security 필터에서 막히지 않게 한다.
3. STOMP 서버 설정 dev-hunt-server/src/main/java/com/cj/devhunt/chat/config/WebSocketConfig.java STOMP endpoint, publish prefix, subscribe broker prefix를 정의한다.
4. 메시지 DTO dev-hunt-server/src/main/java/com/cj/devhunt/chat/presentation/dto/ChatLevel1PingRequest.java
dev-hunt-server/src/main/java/com/cj/devhunt/chat/presentation/dto/ChatLevel1PongResponse.java 		프론트가 보내는 ping 요청과 서버가 내려주는 pong 응답 구조를 정의한다.
5. 메시지 처리 dev-hunt-server/src/main/java/com/cj/devhunt/chat/presentation/ChatStudyMessageController.java /app/chat.level1.ping으로 들어온 메시지를 받고 /topic/chat.level1.pong으로 응답한다.
6. 프론트 STOMP 패키지 dev-hunt-frontend/package.json @stomp/stompjs 클라이언트 사용. 브라우저에서 STOMP 연결, 구독, 발행을 처리한다.
7. Level 1 라우팅 dev-hunt-frontend/src/app/chat-study/[level]/page.tsx level-1일 때 실제 STOMP 테스트 화면인 ChatStudyLevel1Page를 렌더링한다.
8. 테스트 화면 dev-hunt-frontend/src/features/chat-study-level1/ChatStudyLevel1Page.tsx 연결, 연결 해제, ping 전송, pong 수신 로그를 담당하는 Level 1 핵심 화면.
9. 메뉴 연결 dev-hunt-server/src/main/java/com/cj/devhunt/config/MenuSeeder.java
dev-hunt-frontend/src/widgets/header/ui/Header.tsx 		메뉴에서 채팅 > 채팅 스터디 > Level 1로 접근할 수 있게 연결한다.

백엔드 로직

1. WebSocketConfig

STOMP 연결의 기준 경로와 메시지 prefix를 정한다.

@Configuration
@EnableWebSocketMessageBroker
public class WebSocketConfig implements WebSocketMessageBrokerConfigurer {

    @Override
    public void registerStompEndpoints(StompEndpointRegistry registry) {
        registry.addEndpoint("/ws")
                .setAllowedOriginPatterns("http://localhost:*", "http://127.0.0.1:*");
    }

    @Override
    public void configureMessageBroker(MessageBrokerRegistry registry) {
        registry.setApplicationDestinationPrefixes("/app");
        registry.enableSimpleBroker("/topic", "/queue");
        registry.setUserDestinationPrefix("/user");
    }
}
  • /ws: 브라우저가 접속하는 WebSocket endpoint
  • /app: 클라이언트가 서버 메서드로 메시지를 보낼 때 붙이는 prefix
  • /topic: 여러 클라이언트가 구독하는 broadcast 목적지
  • /queue: 사용자별 또는 1:1 메시지로 확장할 때 사용할 목적지

2. ChatStudyMessageController

클라이언트 publish 메시지를 받아 서버 응답을 topic으로 내보낸다.

@MessageMapping("/chat.level1.ping")
@SendTo("/topic/chat.level1.pong")
public ChatLevel1PongResponse ping(ChatLevel1PingRequest request) {
    return new ChatLevel1PongResponse(
            sender,
            "pong: " + message,
            Instant.now()
    );
}
  • 프론트 발행 주소: /app/chat.level1.ping
  • 실제 매핑 주소: /chat.level1.ping
  • 응답 구독 주소: /topic/chat.level1.pong

프론트 로직

1. 연결 URL 생성

NEXT_PUBLIC_API_URL이 있으면 그 값을 사용하고, 없으면 http://localhost:4101을 기본값으로 사용한다.

function getWebSocketUrl() {
  const apiUrl = process.env.NEXT_PUBLIC_API_URL ?? "http://localhost:4101";
  return `${apiUrl.replace(/^http/, "ws").replace(/\/$/, "")}/ws`;
}

2. STOMP Client 생성

연결 버튼을 누르면 @stomp/stompjs의 Client를 생성하고 활성화한다.

const client = new Client({
  brokerURL: wsUrl,
  reconnectDelay: 0,
  onConnect: () => {
    client.subscribe("/topic/chat.level1.pong", callback);
  },
});

3. Ping 발행

연결된 상태에서 버튼을 누르면 서버의 @MessageMapping으로 메시지를 보낸다.

client.publish({
  destination: "/app/chat.level1.ping",
  body: JSON.stringify({
    sender,
    message,
  }),
});

테스트 절차

  1. 백엔드 서버를 재시작한다. 새 의존성과 WebSocket 설정은 재시작 후 반영된다.
  2. 프론트에서 http://localhost:4300/chat-study/level-1/로 이동한다.
  3. 연결 버튼을 누른다.
  4. 상태가 연결됨으로 바뀌고 이벤트 로그에 연결 완료가 찍히는지 확인한다.
  5. 메시지를 입력하고 Ping 보내기를 누른다.
  6. 로그에 sent와 received가 모두 찍히는지 확인한다.
  7. 연결 해제 버튼을 누르고 상태가 끊김으로 바뀌는지 확인한다.

Level 1에서 확인하려는 것

  • Spring Boot WebSocket starter가 정상 동작하는지
  • STOMP endpoint /ws로 브라우저가 연결되는지
  • /app으로 보낸 메시지가 @MessageMapping에 도착하는지
  • /topic 구독자가 서버 응답을 받는지
  • 프론트에서 연결 상태와 에러를 눈으로 확인할 수 있는지
