STOMP 연결 객체와 clientRef 설명

이 문서는 new Client(...)로 만들어지는 연결 객체가 무엇인지, 그리고 왜 clientRef.current = client로 저장해야 하는지 정리한다.

핵심 결론

client는 STOMP 연결을 제어하는 객체다. 연결 시작, 구독, 메시지 전송, 연결 해제를 모두 이 객체로 처리한다.

clientRef.current = client는 이 연결 객체를 React 컴포넌트가 계속 기억하게 저장하는 코드다. 저장하지 않으면 연결 해제, ping 전송, 중복 연결 방지, unmount 정리를 안정적으로 할 수 없다.

연결 객체란?

const client = new Client({
  brokerURL: wsUrl,
  onConnect: () => {
    client.subscribe("/topic/chat.level1.pong", callback);
  },
});

여기서 client는 실제 WebSocket 연결 그 자체라기보다, WebSocket 위에서 STOMP 프로토콜을 제어하는 컨트롤러 객체에 가깝다.

이 객체가 할 수 있는 대표 작업은 다음과 같다.

메서드/속성 역할
client.activate() WebSocket/STOMP 연결을 시작한다.
client.deactivate() 연결을 종료한다.
client.subscribe(destination, callback) 특정 topic/queue를 구독하고, 메시지가 오면 callback을 실행한다.
client.publish(...) 서버로 메시지를 발행한다.
client.connected 현재 STOMP 연결이 완료된 상태인지 확인한다.
client.active client가 활성 상태인지, 또는 연결 시도 중인지 확인한다.

new Client는 연결이 아니다

new Client({...})는 연결을 바로 여는 코드가 아니다. 연결에 필요한 설정과 콜백을 가진 객체를 만드는 단계다. 

const client = new Client({...}); // 설정 객체 생성

client.activate();               // 실제 연결 시작
연결 성공 여부는 await로 받는 것이 아니라 onConnect, onWebSocketError, onStompError 같은 이벤트 콜백으로 처리한다.

왜 일반 변수로 두면 안 되나?

만약 이렇게만 쓰면:

const connect = () => {
  const client = new Client({...});
  client.activate();
};

client는 connect() 함수 안에서만 접근 가능한 지역 변수다. 함수가 끝나면 다른 함수에서는 이 객체를 직접 사용할 수 없다.

그 결과 다음 작업이 어려워진다.

  • 연결 해제 버튼에서 같은 client를 찾아 deactivate() 호출
  • Ping 버튼에서 같은 client를 찾아 publish() 호출
  • 이미 연결 중인지 확인해서 중복 연결 방지
  • 페이지를 벗어날 때 기존 연결 정리

그래서 clientRef에 저장한다

const clientRef = useRef<Client | null>(null);

const connect = () => {
  const client = new Client({...});

  clientRef.current = client;
  client.activate();
};

useRef는 React 컴포넌트가 렌더링을 다시 해도 값을 유지할 수 있는 저장소다. clientRef.current에 client를 넣어두면 다른 함수에서도 같은 연결 객체를 꺼내 쓸 수 있다.

clientRef.current = client의 의미

clientRef.current = client;

이 한 줄의 의미는 다음과 같다.

  • 방금 만든 STOMP client 객체를 컴포넌트 전역 보관소에 저장한다.
  • 다른 함수에서 clientRef.current로 같은 객체를 다시 꺼낼 수 있게 한다.
  • 렌더링이 다시 되어도 client 객체를 잃어버리지 않게 한다.
쉽게 말하면 const client = new Client(...)는 연결 도구를 만드는 것이고, clientRef.current = client는 그 도구를 계속 쓸 수 있게 보관함에 넣는 것이다.

clientRef가 필요한 실제 코드들

1. 중복 연결 방지

if (clientRef.current?.active) return;

이미 client가 활성 상태라면 새 client를 만들지 않는다. 사용자가 연결 버튼을 여러 번 눌러도 WebSocket 연결이 중복 생성되지 않게 막는다.

2. 연결 해제

const disconnect = async () => {
  const client = clientRef.current;
  if (!client) return;

  await client.deactivate();
  clientRef.current = null;
};

연결 해제 버튼을 누를 때도 처음 만든 같은 client 객체를 찾아야 한다. 그래서 clientRef.current에서 꺼내 deactivate()를 호출한다.

3. Ping 전송

const sendPing = () => {
  const client = clientRef.current;
  if (!client?.connected) return;

  client.publish({
    destination: "/app/chat.level1.ping",
    body: JSON.stringify(payload),
  });
};

메시지를 보낼 때도 같은 STOMP client가 필요하다. 연결된 client가 있어야 서버로 publish할 수 있다.

4. 컴포넌트 종료 시 정리

useEffect(() => {
  return () => {
    void clientRef.current?.deactivate();
  };
}, []);

사용자가 페이지를 벗어나면 기존 WebSocket 연결을 닫아야 한다. ref에 저장해두면 unmount cleanup에서도 같은 client를 찾아 정리할 수 있다.

왜 useState가 아니라 useRef인가?

구분 useState useRef
값 변경 시 렌더링 다시 렌더링됨 다시 렌더링되지 않음
화면 표시용 데이터 적합 부적합
외부 객체 보관 가능하지만 불필요한 렌더링이 생길 수 있음 적합
STOMP client 저장 권장하지 않음 권장

STOMP client 객체는 화면에 직접 보여줄 값이 아니다. 연결 상태는 useState로 관리하고, 실제 연결 객체는 useRef로 보관하는 것이 자연스럽다.

상태와 객체를 분리해서 생각하기

connectionState

화면에 보여줄 상태다.

idle, connecting, connected, disconnected, error

clientRef.current

실제 STOMP 연결을 제어하는 객체다.

activate, deactivate, subscribe, publish

둘은 역할이 다르다.

  • connectionState: 사용자에게 현재 상태를 보여주기 위한 값
  • clientRef.current: 실제 연결을 조작하기 위한 객체

중복 생성 방지 흐름

  1. 처음 연결 버튼 클릭
  2. clientRef.current는 null이므로 통과
  3. new Client(...)로 client 생성
  4. clientRef.current = client로 저장
  5. client.activate()로 연결 시작
  6. 다시 연결 버튼 클릭
  7. clientRef.current?.active가 true라서 return
  8. 새 client가 만들어지지 않음

잘못된 예시

연결할 때마다 새 client를 만들고 저장하지 않는 경우

const connect = () => {
  const client = new Client({...});
  client.activate();
};

const sendPing = () => {
  // 여기서는 위 client를 알 수 없다.
};

이 구조에서는 sendPing, disconnect에서 같은 client를 사용할 수 없다. 또한 연결 버튼을 누를 때마다 새 client가 만들어질 수 있다.

현재 구조

const clientRef = useRef<Client | null>(null);

const connect = () => {
  if (clientRef.current?.active) return;

  const client = new Client({...});
  clientRef.current = client;
  client.activate();
};

const sendPing = () => {
  const client = clientRef.current;
  if (!client?.connected) return;
  client.publish(...);
};

const disconnect = async () => {
  const client = clientRef.current;
  if (!client) return;
  await client.deactivate();
  clientRef.current = null;
};

이 구조는 연결 객체를 하나의 ref에 보관하고, 필요한 함수들이 같은 객체를 공유한다. React에서 WebSocket, STOMP, 타이머, 외부 라이브러리 인스턴스를 다룰 때 흔히 쓰는 방식이다.

정리

  • client는 STOMP 연결을 제어하는 객체다.
  • new Client(...)는 객체 생성이고, 실제 연결은 client.activate()다.
  • clientRef.current = client는 만든 client를 계속 쓰기 위해 저장하는 코드다.
  • useRef는 렌더링과 무관하게 외부 객체를 보관하기 좋다.
  • 중복 연결 방지, 연결 해제, 메시지 전송, unmount 정리에 모두 clientRef.current가 필요하다.
