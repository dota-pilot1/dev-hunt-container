WebSocket STOMP 핵심 흐름

이 문서는 WebSocket 연결, STOMP CONNECT, 연결 유지 중 publish/subscribe, 그리고 disconnect까지의 전체 흐름을 정리한다.

전체 그림

1. WebSocket 연결 │ ▼ ws://localhost:4101/ws 2. STOMP CONNECT │ ▼ 연결 유지 (CONNECTED) │ ├── subscribe("/topic/...") ├── publish("/app/...") ├── unsubscribe() ├── publish("/app/...") ├── publish("/app/...") └── ... 마지막 │ ▼ disconnect()
예시에서 포트는 현재 프로젝트 기준 4101이다. 일반 Spring Boot 기본 포트 8080을 쓰는 프로젝트라면 ws://localhost:8080/ws가 된다.

핵심 결론

/ws는 WebSocket 연결을 여는 입구다. 연결이 열린 뒤 실제 채팅 기능은 STOMP 명령인 CONNECT, SUBSCRIBE, SEND, UNSUBSCRIBE, DISCONNECT로 처리된다.

즉 연결 URL은 하나지만, 연결 안에서 여러 destination을 사용해 기능을 나눈다. 

연결 URL
  ws://localhost:4101/ws

메시지 목적지
  /app/chat.level1.ping
  /topic/chat.level1.pong
  /topic/chat.rooms.{roomId}
  /user/queue/notifications

1. WebSocket 연결

첫 단계는 브라우저와 서버 사이에 WebSocket 통신 통로를 여는 것이다. 이때 사용하는 주소가 ws://localhost:4101/ws다.

구분 코드 의미
백엔드 registry.addEndpoint("/ws") WebSocket 접속 endpoint를 연다.
프론트 brokerURL: wsUrl ws://localhost:4101/ws로 접속한다.
실행 client.activate() 실제 연결을 시작한다.

이 단계에서 연결되는 것은 아직 “채팅방 구독”이 아니다. 서버와 브라우저 사이에 실시간 통신이 가능한 통로를 여는 단계다.

2. STOMP CONNECT

WebSocket 연결이 열리면 그 위에서 STOMP 프로토콜 연결을 시작한다. @stomp/stompjs는 내부적으로 STOMP CONNECT 프레임을 보낸다. 

CONNECT
accept-version:1.2
heart-beat:10000,10000

서버가 이를 받아들이면 CONNECTED 프레임을 응답하고, 프론트의 onConnect 콜백이 실행된다. 

onConnect: () => {
  setConnectionState("connected");
  client.subscribe("/topic/chat.level1.pong", callback);
}
onConnect가 실행됐다는 것은 WebSocket 연결뿐 아니라 STOMP 세션까지 준비됐다는 뜻이다.

3. 연결 유지 상태

CONNECTED 이후에는 같은 WebSocket 연결을 계속 유지하면서 여러 작업을 할 수 있다.

subscribe()

서버가 보내는 특정 topic/queue 메시지를 받겠다고 등록한다.

publish()

서버로 메시지를 보낸다. STOMP 프레임 기준으로는 SEND다.

unsubscribe()

더 이상 특정 topic/queue 메시지를 받지 않겠다고 해제한다.

disconnect()

STOMP 연결과 WebSocket 연결을 정리한다.

4. subscribe()

subscribe()는 서버 Controller를 호출하는 코드가 아니다. broker에게 “이 destination으로 오는 메시지를 나에게 보내줘”라고 등록하는 코드다. 

client.subscribe("/topic/chat.level1.pong", (frame) => {
  const body = JSON.parse(frame.body);
  console.log(body);
});

내부적으로는 이런 STOMP 프레임을 보낸다.

SUBSCRIBE
destination:/topic/chat.level1.pong
id:sub-0

서버의 simple broker는 현재 WebSocket 세션을 /topic/chat.level1.pong 구독자로 등록한다.

5. publish()

publish()는 서버로 메시지를 보내는 코드다. @stomp/stompjs의 API 이름은 publish지만, STOMP 프레임 기준으로는 SEND다. 

client.publish({
  destination: "/app/chat.level1.ping",
  body: JSON.stringify({
    sender: "tester",
    message: "hello websocket stomp",
  }),
});

내부적으로는 이런 STOMP 프레임을 보낸다.

SEND
destination:/app/chat.level1.ping
content-type:application/json

{"sender":"tester","message":"hello websocket stomp"}

백엔드에서는 /app prefix가 제거되고, @MessageMapping("/chat.level1.ping") 메서드가 실행된다.

6. 서버 응답과 topic 전달

서버는 @MessageMapping으로 메시지를 받은 뒤, @SendTo에 지정된 topic으로 응답을 보낸다. 

@MessageMapping("/chat.level1.ping")
@SendTo("/topic/chat.level1.pong")
public ChatLevel1PongResponse ping(ChatLevel1PingRequest request) {
    return new ChatLevel1PongResponse(sender, "pong: " + message, Instant.now());
}

그러면 simple broker가 /topic/chat.level1.pong을 구독 중인 모든 프론트 세션에 메시지를 전달한다.

7. unsubscribe()

unsubscribe()는 특정 topic/queue 구독을 해제한다. Level 1 코드에서는 아직 직접 쓰지 않지만, 방을 이동하거나 특정 채널 수신을 끊을 때 필요하다. 

const subscription = client.subscribe("/topic/chat.level1.pong", callback);

// 나중에 구독 해제
subscription.unsubscribe();

채팅방 기능으로 확장하면 방을 나갈 때 해당 방 topic 구독을 해제하는 방식으로 사용한다.

8. disconnect()

마지막 단계는 연결을 정리하는 것이다. 현재 코드에서는 client.deactivate()를 사용한다. 

const disconnect = async () => {
  const client = clientRef.current;
  if (!client) return;

  await client.deactivate();
  clientRef.current = null;
  setConnectionState("disconnected");
};

deactivate()는 STOMP 연결을 종료하고 내부 WebSocket도 닫는다.

publish와 subscribe 순서

보통은 먼저 subscribe()를 해두고, 그 다음 publish()를 한다. 그래야 서버가 응답을 topic으로 보냈을 때 프론트가 이미 받을 준비가 되어 있다. 

onConnect: () => {
  client.subscribe("/topic/chat.level1.pong", callback);
};

// 사용자가 버튼 클릭
client.publish({
  destination: "/app/chat.level1.ping",
  body: JSON.stringify(payload),
});

정리

  • ws://localhost:4101/ws는 연결 입구다.
  • client.activate()가 WebSocket 연결과 STOMP CONNECT를 시작한다.
  • onConnect 이후부터 publish/subscribe를 안정적으로 사용할 수 있다.
  • subscribe()는 메시지를 받겠다고 broker에 등록하는 작업이다.
  • publish()는 서버로 메시지를 보내는 작업이다.
  • disconnect() 또는 deactivate()는 연결을 정리하는 작업이다.
  • 연결 URL은 하나이고, 실제 기능 구분은 STOMP destination으로 한다.
