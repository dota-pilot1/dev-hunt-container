Level 2

메시지 컨트롤러 요청 응답 도식화

@MessageMapping과 @SendTo가 붙은 컨트롤러가 프론트 메시지 요청을 받아 구독 중인 클라이언트들에게 응답하는 구조를 간단히 정리한다.

한 줄로 말하면

이 컨트롤러는 /app/chat.level2.send로 들어온 메시지 요청을 받고, 응답 DTO를 만들어 /topic/chat.level2.messages를 구독한 모든 클라이언트에게 브로드캐스트한다.

컨트롤러 코드

@MessageMapping("/chat.level2.send")
@SendTo("/topic/chat.level2.messages")
public ChatLevel2MessageResponse sendMessage(ChatLevel2MessageRequest request) {
    String sender = request.sender() == null || request.sender().isBlank()
            ? "anonymous"
            : request.sender().trim();
    String content = request.content() == null || request.content().isBlank()
            ? "(empty)"
            : request.content().trim();

    return new ChatLevel2MessageResponse(UUID.randomUUID().toString(), sender, content, Instant.now());
}

도식화

1. 프론트 publish 사용자가 보내기 버튼을 누르면 프론트가 /app/chat.level2.send로 메시지를 보낸다.
↓
2. Spring STOMP 라우팅 /app prefix를 보고 application handler로 넘긴다.
↓
3. @MessageMapping 실행 @MessageMapping("/chat.level2.send") 메서드가 요청 body를 ChatLevel2MessageRequest로 받아 처리한다.
↓
4. 응답 DTO 생성 서버가 id, sender, content, sentAt을 담은 ChatLevel2MessageResponse를 반환한다.
↓
5. @SendTo 브로드캐스트 반환값이 /topic/chat.level2.messages로 전송된다.
↓
6. 구독 클라이언트 수신 이 topic을 client.subscribe()로 구독해둔 모든 브라우저의 콜백이 실행된다.

주소 역할 비교

주소 방향 역할 관련 코드
/app/chat.level2.send 프론트 → 백엔드 사용자가 보낸 메시지 요청이 들어가는 주소 client.publish(), @MessageMapping
/topic/chat.level2.messages 백엔드 → 프론트들 서버 응답이 구독 클라이언트들에게 뿌려지는 주소 @SendTo, client.subscribe()

프론트 수신 코드

subscriptionRef.current = client.subscribe("/topic/chat.level2.messages", (frame: IMessage) => {
  const body = JSON.parse(frame.body) as ChatMessage;
  setMessages((prev) => [...prev, body].slice(-100));
});
정리하면, @MessageMapping은 요청을 받는 입구이고 @SendTo는 구독자들에게 응답을 내보내는 출구다. 그래서 이 컨트롤러는 “메시지 요청을 받아 구독한 클라이언트에 응답하는 컨트롤러”라고 보면 된다.
