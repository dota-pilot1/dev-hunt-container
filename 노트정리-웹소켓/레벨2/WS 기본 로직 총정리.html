WebSocket + STOMP

WS 기본 로직 총정리

WebSocket 연결 객체를 만들고, STOMP 규칙으로 요청과 응답을 주고받는 기본 로직을 한 번에 정리한다. Level 2 기준으로 서버 설정, 프론트 연결, 구독, 메시지 발행, 응답 처리까지 이어진다.

전체 요약

웹소켓은 연결 통로를 만들고, STOMP는 그 통로 위에서 메시지를 어느 주소로 보낼지 정하는 규칙이다. 프론트는 Client 객체로 서버에 연결하고, topic을 구독해둔 뒤, 메시지를 application 주소로 발행한다. 서버는 요청을 받아 응답 DTO를 만들고, 구독 topic으로 다시 브로드캐스트한다.

전체 흐름 도식

1. 서버가 WebSocket/STOMP 설정을 연다 /ws endpoint, /app 요청 prefix, /topic 응답 broker를 설정한다.
↓
2. 프론트가 STOMP client 객체를 만든다 new Client({ brokerURL: wsUrl, ... })로 웹소켓 연결 객체를 준비한다.
↓
3. 프론트가 연결한다 client.activate()로 ws://localhost:4101/ws에 연결한다.
↓
4. 연결 성공 후 topic을 구독한다 client.subscribe("/topic/chat.level2.messages", callback)으로 응답 수신 콜백을 등록한다.
↓
5. 사용자가 메시지를 보낸다 client.publish({ destination: "/app/chat.level2.send", body })로 서버에 요청한다.
↓
6. 서버 컨트롤러가 요청을 받는다 @MessageMapping("/chat.level2.send") 메서드가 요청 DTO를 받는다.
↓
7. 서버가 구독 topic으로 응답한다 @SendTo("/topic/chat.level2.messages")로 구독 클라이언트들에게 응답 DTO를 보낸다.
↓
8. 프론트 구독 콜백이 응답을 처리한다 JSON.parse(frame.body) 후 setMessages()로 화면 목록에 추가한다.

웹소켓 객체로 요청과 응답을 주고받는다

프론트에서 쓰는 핵심 객체는 @stomp/stompjs의 Client다. 이 객체 하나가 연결, 구독, 발행을 모두 담당한다. 

const client = new Client({
  brokerURL: wsUrl,
  reconnectDelay: 0,
  onConnect: () => {
    // 연결 성공 후 구독 등록
  },
  onStompError: (frame) => {
    // STOMP 에러 처리
  },
  onWebSocketError: () => {
    // WebSocket 에러 처리
  },
});
메서드 역할
client.activate() 서버의 WebSocket endpoint로 실제 연결을 시작한다.
client.subscribe(topic, callback) 서버가 특정 topic으로 보내는 응답을 받도록 콜백을 등록한다.
client.publish({ destination, body }) 서버의 application destination으로 메시지를 보낸다.
client.deactivate() 연결을 종료한다.

서버 핵심 설정

서버에서 가장 중요한 설정은 WebSocketConfig다. 여기서 연결 주소와 STOMP 메시지 주소 규칙이 결정된다. 

@Configuration
@EnableWebSocketMessageBroker
public class WebSocketConfig implements WebSocketMessageBrokerConfigurer {

    @Override
    public void registerStompEndpoints(StompEndpointRegistry registry) {
        registry.addEndpoint("/ws")
                .setAllowedOriginPatterns("http://localhost:*", "http://127.0.0.1:*");
    }

    @Override
    public void configureMessageBroker(MessageBrokerRegistry registry) {
        registry.setApplicationDestinationPrefixes("/app");
        registry.enableSimpleBroker("/topic", "/queue");
        registry.setUserDestinationPrefix("/user");
    }
}
설정 의미
addEndpoint("/ws") 프론트가 연결할 WebSocket endpoint다. 최종 연결 URL은 ws://localhost:4101/ws가 된다.
setApplicationDestinationPrefixes("/app") 프론트가 서버 컨트롤러로 메시지를 보낼 때 사용하는 prefix다.
enableSimpleBroker("/topic", "/queue") 서버가 클라이언트들에게 메시지를 뿌리는 broker prefix다.

프론트에서 이렇게 연결한다

먼저 API 주소를 WebSocket 주소로 바꾼다. http://localhost:4101이면 ws://localhost:4101/ws가 된다. 

function getWebSocketUrl() {
  const apiUrl = process.env.NEXT_PUBLIC_API_URL ?? "http://localhost:4101";
  return `${apiUrl.replace(/^http/, "ws").replace(/\/$/, "")}/ws`;
}

그리고 connect()에서 STOMP client를 만들고 활성화한다. 

const connect = () => {
  if (clientRef.current?.active) return;

  setConnectionState("connecting");

  const client = new Client({
    brokerURL: wsUrl,
    reconnectDelay: 0,
    onConnect: () => {
      setConnectionState("connected");
      // 여기서 구독 등록
    },
  });

  clientRef.current = client;
  client.activate();
};
clientRef.current에 저장하는 이유는 렌더링과 무관하게 같은 STOMP client 객체를 계속 들고 있어야 하기 때문이다. 메시지를 보낼 때도 이 ref에서 client를 꺼내 사용한다.

구독하고

연결 성공 후에는 서버 응답을 받을 topic을 구독한다. 이 코드는 “지금 메시지를 보내라”가 아니라 “앞으로 이 주소로 오는 메시지를 받겠다”는 등록이다. 

subscriptionRef.current = client.subscribe("/topic/chat.level2.messages", (frame: IMessage) => {
  const body = JSON.parse(frame.body) as ChatMessage;
  setMessages((prev) => [...prev, body].slice(-100));
});

서버가 /topic/chat.level2.messages로 응답을 보내면 이 콜백이 실행된다. 콜백은 응답 body를 JSON으로 파싱하고, messages state에 추가한다.

메시지 보내고

사용자가 보내기 버튼을 누르면 sendMessage()가 실행된다. 여기서는 구독 주소가 아니라 서버 컨트롤러로 들어가는 /app/chat.level2.send로 보낸다. 

const sendMessage = (event?: FormEvent<HTMLFormElement>) => {
  event?.preventDefault();

  const client = clientRef.current;
  const content = draft.trim();
  if (!client?.connected || !content) return;

  client.publish({
    destination: "/app/chat.level2.send",
    body: JSON.stringify({ sender, content }),
  });
  setDraft("");
};

보내는 JSON

{
  "sender": "오현석",
  "content": "안녕하세요"
}

서버 컨트롤러가 요청 받고 응답한다

프론트가 /app/chat.level2.send로 보낸 메시지는 서버의 @MessageMapping("/chat.level2.send") 메서드로 들어온다. 메서드가 반환한 값은 @SendTo에 적힌 topic으로 전송된다. 

@MessageMapping("/chat.level2.send")
@SendTo("/topic/chat.level2.messages")
public ChatLevel2MessageResponse sendMessage(ChatLevel2MessageRequest request) {
    String sender = request.sender() == null || request.sender().isBlank()
            ? "anonymous"
            : request.sender().trim();
    String content = request.content() == null || request.content().isBlank()
            ? "(empty)"
            : request.content().trim();

    return new ChatLevel2MessageResponse(
            UUID.randomUUID().toString(),
            sender,
            content,
            Instant.now()
    );
}
어노테이션 역할
@MessageMapping("/chat.level2.send") 프론트가 /app/chat.level2.send로 보낸 요청을 받는 입구다.
@SendTo("/topic/chat.level2.messages") 메서드 반환값을 구독 중인 클라이언트들에게 보내는 출구다.

응답은 이렇게 처리

서버가 응답을 topic으로 보내면, 프론트에서 미리 등록해둔 구독 콜백이 실행된다. 이때 화면에 추가되는 것은 입력창이 아니라 messages 목록이다. 

const body = JSON.parse(frame.body) as ChatMessage;
setMessages((prev) => [...prev, body].slice(-100));

응답 JSON

{
  "id": "cc7ae965-3e5c-4f40-b7db-08af9a5f77d6",
  "sender": "오현석",
  "content": "안녕하세요",
  "sentAt": "2026-06-27T10:52:28.741791Z"
}

setMessages()로 state가 바뀌면 React가 다시 렌더링하고, messages.map(...) 부분에서 새 말풍선이 화면에 표시된다.

주소 두 개만 구분하면 된다

주소 방향 설명
/app/chat.level2.send 프론트 → 서버 사용자가 메시지를 보낼 때 publish하는 요청 주소다.
/topic/chat.level2.messages 서버 → 프론트들 서버가 응답을 브로드캐스트하고, 프론트가 subscribe하는 응답 주소다.
최종 정리: 프론트는 Client 웹소켓 객체로 서버에 연결한다. 연결 후 /topic/chat.level2.messages를 구독해 응답 통로를 열어둔다. 사용자가 메시지를 보내면 /app/chat.level2.send로 publish한다. 서버 컨트롤러는 요청을 받아 응답 DTO를 만들고 @SendTo로 topic에 뿌린다. 프론트 구독 콜백은 그 응답을 받아 messages state에 추가하고 화면을 다시 그린다.
